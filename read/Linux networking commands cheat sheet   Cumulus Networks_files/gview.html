SYSTEM ADMINISTRATION

sudo [command]

Become root for one command

man [program]

Display help menu for program

htop

View real-time CPU/memory info

net show system

Show system model number

cat /etc/image-release

Show precise software version

uptime

Display uptime info

free -m

Show free memory in MB

ps aux

Show all running processes

date

Show the current date/time

kill [PID]

Halt process by process-id

cl-license -i [http:// or license.txt]

License a switch

sudo reboot

Reboot the switch immediately

history

Display recently entered commands

sudo systemctl [start|stop|restart|reload] [program].service

Ex. sudo systemctl start quagga.service

Control current execution of a service

sudo systemctl [enable|disable] [program].service

Ex. sudo systemctl enable quagga.service

Start a service on boot

sudo apt-get update

Update the list of available packages

sudo apt-get install [package name]

Install a package from the repository

sudo apt-get upgrade

Update software to latest versions

sudo apt-cache search [string]

Look for packages containing string

sudo apt-cache show [package name]

Show all info on package

sudo snapper create -d “[Snapshot Description]”

Create a snapshot

sudo snapper list

List all snapshots

Using Linux for networking completely unifies the network stack and can often be done by leveraging

existing talent and skills. However many network engineers are accustomed to the commands and

configurations of traditional network operating systems. Making the switch is easy, but it is a transition.

This handy cheat sheet was designed for quick references as you learn and leverage Cumulus Linux. It

features the most common commands used in Cumulus Linux with explanations on how to use them.

TROUBLESHOOTING COMMANDS

arp

Display the ARP table

ping [hostname or IP Address]

Ping a destination

ping -I [Src IP] [Dest hostname or IP]

Ping from a source

ip [-6] route show

Show kernel routing table

ip route show [x.x.x.x/CIDR Mask]

Show kernel routing table for prefix

vrf show

List configured VRFs

sudo cl-acltool -L

Show ACL rules

ip route show table [vrf-name]

List routes in a VRF

sudo cl-acltool -L

Show ACL rules

nslookup [hostname]

Perform DNS lookup

tcpdump -i [swp1]

Collect traffic from control plane

cl-netstat -c

Show interface counters for all interfaces

ethtool -S [interface]

Show port statistics

sudo lldpctl [interface]

Show LLDP neighbors

sudo ptmctl

Show cabling compared to LLDP

(requires a /etc/ptm.d/topology.dot file to be created)

net show bridge macs

Show all MACs on a bridge

bridge vlan show

Show VLAN Membership

clagctl -v

Show MLAG state

vtysh

Login to Quagga

vtysh -c “show run”

Run a single command in Quagga

net show bridge spanning-tree

Show detailed spanning tree state for all interfaces

cat /var/log/syslog

Show system logs

smonctl -v

Show PSU/FAN/Temp info

WORKING WITH FILES

Everything in Linux is a file, so knowing how to work with files is

a very important skillset to master.

nano [filename]

Edit a text file

tree

Show filesystem hierarchy

pwd

Print current folder name

mkdir [folder]

Make new folder

cd [folder]

Change into different folder

ls -lha

List the files in the current folder

mv [old file] [new file]

Move/rename a file

cp [old file] [new file]

Copy a file

cat [filename]

Display a text file

grep “string” [filename]

Search file for a text string

head -n 5 [filename]

Display top 5 lines of a file

tail -n 5 [filename]

Display last 5 lines of a file

tail -f [filename]

Follow file and display new lines

find / -name [filename]

Find file named filename

find / -name “*[string]*”

Find filename containing string

chmod 777 [filename]

Change file permissions

chown [username] [filename]

Change file ownership

REDIRECTION

ip link show > [newfile]

Redirect standard output to a file

ifreload -a &> [newfile]

Redirect standard output and standard error to a file

cat /var/log/syslog | grep kernel

Use one program’s output as another program’s input

[command] &

Send command execution to the background

(use “fg” to send command to foreground again)

Blue – Optional Parameter

Green – Required Parameter

ifup [interface]

Show SFP information

ip monitor link

Monitor changes to link state in real time
